# “吃点啥”随机选餐网页开发指令

## 项目概述
使用 React + TypeScript + Tailwind CSS 开发的移动端优先的周边餐饮随机抽取网页。

## 开发规范
- 统一使用函数式组件 (Functional Components) + Hooks 进行开发
- 严格使用 Tailwind CSS 编写样式，尽量避免手写原生 CSS
- 保持组件的模块化和可复用性（如抽取动画组件、结果卡片组件需独立拆分）
- 将地图 API 调用和地理位置获取逻辑抽离为独立的 Services 或自定义 Hooks
- 核心交互逻辑（定位权限处理、随机抽取算法）代码必须包含清晰的注释

## 设计要求
- 极简直观的 UI 风格（保持界面清爽，突出“开始抽取”的主按钮）
- 核心按钮尺寸需适合移动端单手操作（拇指热区）
- 必须使用 Framer Motion 添加流畅的抽取动画（如轮盘滚动、卡片翻转或盲盒弹跳效果），增强互动反馈
- 严格确保移动端（Mobile First）完美适配，禁止出现横向滚动条

## 注意事项
- 保持功能克制，不要过度设计，坚决不引入注册/登录等繁琐流程
- 异常处理兜底：必须优雅处理“用户拒绝地理位置授权”或“周边 500 米无餐厅”等异常情况，提供友好的 UI 提示或重试按钮
- Mock 数据：在真实地图 API 接入前，先使用本地 Mock 数据（如 10 家虚拟餐厅）跑通核心抽取和动画逻辑
- 性能优化：确保首屏加载极快，动画执行保持高帧率不卡顿
